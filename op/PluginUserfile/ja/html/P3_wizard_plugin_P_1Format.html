Page 3  関数呼出しフォーマット (1st Level Analysis)
 

このページでは1st-Level-Analysisを実行するための関数フォーマットを指定します.
1st-Level-Analysisをプラグインする際には, データの管理を行う関数とは実行の制御を行う関数が必要になります.
ここでは, 作成した1st-Level-Analysisデータの制御を行う関数名を (A)に記述します.
また, 1st-Level-Analysisデータを作成するための関数の呼出し方法を (B)に記述します.

 
 
 

Page 3-1  制御用関数名

 

作成する1st-Level-Analysisデータの データ制御や検索キーの設定, Inner検索キーによるデータ取得方法を 記述するための関数名を(A)に記述します.
この関数は1st-Level-Analysisの実行制御関数と同じディレクトリに作成されます. 検索キーやデータ取得方法の詳細設定を行いたい場合に編集する関数となりますので, この場合に混乱しないような命名をしておくと便利です.

 
  

MATLAB パス上に同一のファイル名を 記述することは出来ません.

 
 
 

Page 3-2  実行フォーマット

 

1st-Level-Analysisデータを作成する際の関数の呼出し方法を (B)のエディットテキストに書き込みます.
この時の使用可能な変数は予約語として定義されています.
またReturn-Valueとして少なくとも, fdata を必要とします.
ここでfhdataが指定されない場合はデフォルトのfhdataが作成されます.

 
 

エラーメッセージ

 

  • Need fdata for return value.
    Return Valueとして fdataが指定されていません.
  • The description of Return-Value is wrong.
    戻り値の変数名が正しく読み取れなかった場合のエラーです.
  • The description of Function's name is lack.
    関数名が読み取れなかった場合のエラーです.
  • No Function to execute.
    実行関数名が読み取れなかった場合のエラーです.
  • Syntax Error : in argument ().
    閉じていない括弧が存在します.

 
 
 

Page 2-3  予約語の説明

 

  • hdata
    Platformデータのヘッダ
    データの種類は状態によります.

  • data
    Platformデータ
    データの種類は状態によります.
    連続データの場合は [time, channel, kind]の3次元データに, 区間データの場合は [block, time, channel, kind]の4次元データになります.

  • fhdata
    1st-Leve-Analysis用のヘッダ.
    ここで検索キー等を指定します.

  • fdata
    1st-Leve-Analysis用のデータ
    データは数値配列という規定以外に特に規定はありませんが, 1st-データの種類は状態によります.
    連続データの場合は [time, channel, kind]の3次元データに, 区間データの場合は [block, time, channel, kind]の4次元データになります.

 
 
 

Page 2-4  Example

  

  • fdata = nan_fcn('mean',data,2);
     ブロックデータに対して平均値をとり1st-Level-Analysisの結果とします.

  • [fdata,fhdata]=my_1st_Ana(hdata,data,a,b);
     引数a,bを持つ独自の1st-Level-Analysis関数.
